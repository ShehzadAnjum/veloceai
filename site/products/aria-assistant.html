What runs today vs. the roadmap

Running today

Real LLM, deeply woven — not bolted on.
  • ✦Two inboxes + WhatsApp, watched 24/7 — Gmail polled every 120s with smart deep-sync; WhatsApp every 10s with JID→contact resolution.
  • ✦LLM classification of every message — intent, a 0–100 urgency score, entities, summary and a proposed due date.
  • ✦Replies in your voice — drafts from a per-contact style profile, and a learning loop that improves from every edit you make.
  • ✦Delegations that close themselves — detects a hand-off, opens and assigns a task, chases the overdue via WhatsApp + a gentle email, and auto-closes when the reply says "done."
  • ✦Approve → send, for real — calendar awareness, WebSocket live updates, audit log, and a 13-page React control panel on the live API.

On the roadmap

From smart drafting to true agency.
  • ◆Agentic chat — wire the already-defined tool_calls schema so "type a command, it executes" actually runs (today it's read-only Q&A).
  • ◆Policy-gated autonomy — "auto-send under $X / below urgency Y," trust lists, confidence thresholds, one-click undo. (Autonomy ships in safe testing-mode today.)
  • ◆Semantic memory — replace keyword/ILIKE retrieval with embeddings for sharper voice consistency.
  • ◆Multi-tenant + secrets hygiene — accounts, per-tenant credential vaults; rotate the keys currently in the repo before any public showcase.

Honesty note: six real LLM workloads run, but the actions are still rule-gated, not an agentic planner. Provider is Gemini + OpenAI.

The role it replaces

An EA and an ops coordinator.

Reading, triaging and drafting across email + WhatsApp (the EA), plus opening, assigning and chasing every delegation to closure (the coordinator) — both running on a 24/7 shift.
